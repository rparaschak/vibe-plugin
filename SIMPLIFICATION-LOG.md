# Simplification Log

Constraints removed from vibe's templates and commands, and what happened after.

**Why this file exists:** instructions accumulate; models improve. A constraint that compensates for a weakness the model no longer has is pure context cost. Practice (from Learn Harness Engineering L4/L12): periodically disable a suspect constraint, observe real runs, and if quality holds — delete it and record it here. An entry is a success, not an apology.

**Entry format:** date | constraint removed | where it lived | why it existed | evidence quality held | follow-up.
