# Composition Standard

Obeyed by: any builder command that composes a project command from a skeleton plus a preset, flow-spec, or loop-spec тАФ currently `build-harness`, `build-flow`, `build-loop`; future builders inherit this contract by reference, never by restating it.

The mandate: two conforming builders, given the same skeleton and the same preset/flow-spec, MUST stamp byte-identical output тАФ no re-authoring, no re-ordering, no spacing drift. The rules below are the letter of that mandate.

## Composition spec (build determinism тАФ apply exactly on every command/flow stamp)
Presets live at `presets/<preset>/*.md`; each supplies the skeleton's fills (`DESCRIPTION`, `OPT_OUTS`, `ROLE_SUMMARY`, `NAMED_ARTIFACTS`) and one FLOW block (between `<!-- FLOW тАж -->` and `<!-- FLOW END -->`), injected into `command-skeleton.md`'s single `{{FLOW}}` slot. Generation-time `HARNESS_ROOT` тИИ {`.claude`, `.grok`} is fixed by build-harness before any stamp. When stamping:
- **W-A** тАФ inject the FLOW block verbatim (no summarizing/re-authoring) but still resolve every embedded `{{PLACEHOLDER}}` inside it at stamp time: `{{SPEC_TEMPLATE_PATH}}`, `{{PLAN_TEMPLATE_PATH}}`, `{{RESEARCH_TEMPLATE_PATH}}` тЖТ the stamped-template paths (`HARNESS_ROOT/templates/<name>.md`); `{{CMD_PLAN}}`, `{{CMD_IMPLEMENT}}`, `{{CMD_SPEC}}` тЖТ `/<cmd>` when `COMMAND_PREFIX` is empty, else `/<COMMAND_PREFIX>:<cmd>` (e.g. `/plan` or `/dev:plan`). "Verbatim" is exemption from re-authoring, not from placeholder resolution.
- **W-B** тАФ every resolved file path substituted into generated text is backtick-wrapped.
- **W-C** тАФ when a FIXED skeleton section is opted out, renumber the remaining `## Outline` steps sequentially тАФ leave no gap at the removed step's old number.
- **W-D** тАФ when `archive` is opted out, drop the exact clause "and never archive a Blocked ledger" тАФ including the joining "and" тАФ from the Finalize ordering line, not only the `archive` arrow item; the resulting text reads exactly "тАФ load-bearing:".
- **W-E** тАФ strip **every** builder-facing annotation from output in ALL cases (opted out or not): the `(Opt-out-able: тАж)` parenthetical, every `<!-- BUILDERтАж -->` / `<!-- FILLтАж -->` / rationale comment, and every `>` blockquote meta-note addressed to the builder (e.g. lines beginning `> Skeleton тАж` that explain how skeleton sections map). These are authoring-time notes, never runtime text.
- **W-F** тАФ `opt-out:` names may target a full section heading **or** a Finalize sub-item (`archive`, `commit`); recognize both levels. Emit the frontmatter `opt-out:` line only when non-empty; omit the line entirely when empty.
- **W-G** тАФ on every stamp, **including doctor re-runs of a section you actually re-emit**, rewrite that file's header `generated <date>` to today. (An idempotent re-run re-emits nothing, so no date moves тАФ see Doctor.)
- **W-H** тАФ emit exactly one blank line immediately above and below the injected `{{FLOW}}` content, regardless of the source file's blank lines.
- **W-I** тАФ **harness-root rewrite.** Templates and presets may still write project-relative harness paths with the historical `.claude/` prefix. After composition (and for every non-command stamp too), rewrite every project-relative path prefix `.claude/` тЖТ `HARNESS_ROOT/` when `HARNESS_ROOT` is not `.claude` (e.g. `.grok/`). Never rewrite `$CLAUDE_PLUGIN_ROOT` / `$GROK_PLUGIN_ROOT`, plugin-install paths, or non-path prose about Claude Code the product. When `HARNESS_ROOT` is `.claude`, W-I is a no-op. Same input + same `HARNESS_ROOT` тЖТ byte-identical output.
- **W-J** тАФ **command-prefix path & invoke.** Generation-time `COMMAND_PREFIX` тИИ {empty} тИк kebab-case slugs (not `vibe`, `local`, `user`, `repo`). Command stamp target: empty тЖТ `HARNESS_ROOT/commands/<cmd>.md`; non-empty тЖТ `HARNESS_ROOT/commands/<COMMAND_PREFIX>/<cmd>.md`. Same for build-flow: `HARNESS_ROOT/commands/[<COMMAND_PREFIX>/]<slug>.md`. Every `{{CMD_*}}` in a composed **command** must be resolved at stamp (see W-A); a bare leftover `{{CMD_*}}` placeholder is a defect. **Preset FLOW / command body** invokers for peer harness commands use `{{CMD_PLAN}}` / `{{CMD_IMPLEMENT}}` / `{{CMD_SPEC}}` тАФ never bare `/plan` / `/implement` / `/spec`. Non-command stamps (workspace doc templates, agent descriptions, constitution seeds) stay **prefix-agnostic**: rephrase with phase language (тАЬbefore implementтАЭ, тАЬthe plan commandтАЭ, тАЬduring specтАЭ) rather than slash invokers or `{{CMD_*}}` (those artifacts are not run through command composition). Same inputs + same `COMMAND_PREFIX` тЖТ byte-identical output.
- **`<named artifacts>`** тАФ the angle-bracket slot in the boundary sentence is distinct from `{{PLACEHOLDER}}`; fill it from the preset's `FILL NAMED_ARTIFACTS`, comma-joined into prose тАФ resolved, not left as a literal token.
- **Regen stamp** тАФ fill the `<!-- vibe:regen preset={{PRESET}} ┬╖ flow-spec={{FLOW_SPEC}} -->` line per builder, spelling out the inverse literally: build-harness stamps `preset=<preset-dir> ┬╖ flow-spec=` (empty). build-flow stamps `preset= ┬╖ flow-spec=<HARNESS_ROOT/flows/<slug>.md>` (empty preset). Never `n-a` for the empty side тАФ this makes `--regen` re-derivable.
- **Generated-command header** тАФ for a preset stamp, the generated command's `vibe-template:` header names `presets/<preset>/<cmd>.md` and its version тАФ never the skeleton; for a FLOW stamp it names the flow-spec (`HARNESS_ROOT/flows/<slug>.md`). Doctor/drift-diff checks FIXED sections against `templates/skeletons/command-skeleton.md`, and the FLOW middle against the preset file (preset stamp) or the flow-spec's injected sections (FLOW stamp), respectively.
- **W-K** тАФ **loop stamps.** build-loop composes through `templates/skeletons/loop-command-skeleton.md`: every rule above applies unchanged with the `{{STAGES}}` slot in place of `{{FLOW}}` (W-A/W-H read "STAGES" for "FLOW"). Loop-specs live flat at `HARNESS_ROOT/loops/<slug>.md`; the regen stamp is `loop-spec=<that path>` (no `preset=`/`flow-spec=` sides exist on a loop stamp); the generated header names the loop-spec; drift-diff checks FIXED sections against `loop-command-skeleton.md`. Every cli/model/permission-mode literal in the spec's `## Stages` is resolved text, not a placeholder тАФ a leftover `{{WORKER_CLI}}`-style slot in composed output is a defect, same as a bare `{{CMD_*}}`.
